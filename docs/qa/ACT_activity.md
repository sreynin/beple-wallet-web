#### SCN-ACT-001 은행계좌 목록 정상 조회 (Happy Path)

| 항목 | 내용 |
|------|------|
| Feature ID | F-ACT-001 |
| Screen ID | ACT-001 |
| 우선순위 | P0 |

**전제 조건**
- 사용자가 1개 이상의 은행계좌를 등록한 상태

**실행 단계**
1. 계좌 관리 화면(act-accounts.html)으로 진입한다.
2. 계좌 리스트 영역을 확인한다.
3. 각 항목에 은행 로고, 은행명, 계좌번호(마스킹)가 표시되는지 확인한다.
4. 각 항목에 삭제 버튼이 표시되는지 확인한다.
5. 하단에 "계좌 등록하기" 버튼이 표시되는지 확인한다.

**기대 결과**
- 등록된 계좌 목록이 정상 표시된다.
- 계좌번호는 마스킹 처리되어 표시된다.
- "계좌 등록하기" 버튼이 하단에 표시된다.

#### SCN-ACT-003 은행계좌 등록 — 정상 흐름 (Happy Path)

| 항목 | 내용 |
|------|------|
| Feature ID | F-ACT-002 |
| Screen ID | ACT-002 |
| 우선순위 | P0 |

**전제 조건**
- 계좌 관리 화면에서 "계좌 등록하기" 버튼 탭

**실행 단계**
1. 계좌 등록 화면(act-account-reg.html)으로 진입한다.
2. 은행 목록 그리드에서 은행을 선택한다.
3. 계좌번호 TextField에 유효한 계좌번호를 입력한다.
4. ARS 인증 안내 영역에서 전화번호와 안내 문구를 확인한다.
5. "등록하기" 버튼을 탭한다.
6. ARS 인증 전화를 수신하고 인증을 완료한다.

**기대 결과**
- ARS 인증 완료 후 계좌가 정상 등록된다.
- 계좌 목록 화면으로 복귀하며 신규 계좌가 목록에 추가된다.

#### SCN-ACT-004 거래 내역 정상 조회 (Happy Path)

| 항목 | 내용 |
|------|------|
| Feature ID | F-ACT-004 |
| Screen ID | ACT-003 |
| 우선순위 | P0 |

**전제 조건**
- 사용자에게 충전·결제·출금 내역이 존재하는 상태

**실행 단계**
1. 거래 내역 화면(act-history.html)으로 진입한다.
2. 날짜별 그룹 리스트가 표시되는지 확인한다.
3. 각 항목에 아이콘, 거래명, 금액, 상태가 표시되는지 확인한다.
4. 필터 바에서 날짜 범위를 변경한다.
5. 필터 바에서 유형(충전/결제/출금) 칩을 선택한다.
6. 필터 바에서 상태 칩을 선택한다.

**기대 결과**
- 전체 거래 내역이 날짜별로 그룹화되어 표시된다.
- 필터 변경 시 해당 조건에 맞는 내역만 표시된다.

#### SCN-ACT-005 거래 상세 정상 조회 (Happy Path)

| 항목 | 내용 |
|------|------|
| Feature ID | F-ACT-005 |
| Screen ID | ACT-004 |
| 우선순위 | P0 |

**전제 조건**
- 거래 내역 화면에서 특정 거래 항목을 탭

**실행 단계**
1. 거래 상세 화면(act-detail.html)으로 진입한다.
2. 상단 TopBar에 "거래 상세" 타이틀이 표시되는지 확인한다.
3. 완료/취소/대기 상태 배지를 확인한다.
4. 금액이 중앙 대형으로 표시되는지 확인한다.
5. 상세 정보 테이블(거래 유형, 날짜/시간, 수단, 수수료, 처리번호)이 표시되는지 확인한다.
6. 하단 "영수증 저장" 버튼이 표시되는지 확인한다.

**기대 결과**
- 상태 배지, 금액, 상세 정보 테이블이 모두 정상 표시된다.
- "영수증 저장" 버튼이 하단에 노출된다.
